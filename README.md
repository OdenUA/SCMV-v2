# SCMV v2

Лёгкий интерфейс для просмотра и анализа треков устройств (Device Track). Проект использует Leaflet для отображения карт и пользовательские скрипты для работы с треками, стоянками и построением маршрутов.

## Быстрый старт

1. Клонируйте репозиторий (если ещё не склонирован):

   git clone https://github.com/OdenUA/SCMV-v2.git

2. Откройте проект локально в браузере:

   - Простейший способ: открыть `index.html` в браузере (поддерживается как статическая страница).
   - Рекомендуется запускать через локальный HTTP-сервер (чтобы избежать ограничений браузера на запросы/плагины). Например, если у вас установлен Python 3:

     ```powershell
     cd "d:\scmv\SCMV v2"
     python -m http.server 8000
     # затем открыть http://localhost:8000 в браузере
     ```

3. В интерфейсе:
   - Панель слева содержит фильтры по устройству и датам.
   - Кнопка с иконкой 📍 (в левом верхнем углу) включает режим ручного построения маршрута.
   - В режиме маршрута клики по карте, трекам и маркерам добавляют точки маршрута; попапы при этом подавлены.
   - Кнопка ⟳ сбрасывает текущий ручной маршрут.

## Зависимости / используемые библиотеки

- Leaflet (подключается из CDN в `index.html`)
- leaflet-polyline-decorator (подключается из CDN)

Все зависимости подключены напрямую в `index.html`, дополнительных сборок не требуется.

## Важные файлы

- `index.html` — основной HTML-файл и точка входа.
- `style.css` — стили приложения.
- `map.js` — инициализация карты и элементы управления (контролы, переключатели).
- `route.js` — логика ручного построения маршрута (режим маршрута, сбор точек, построение по OSRM и т.д.).
- `device_track.js` — рисование треков, обработка аномалий и клики по трекам.
- `stops.js` — логика стоянок (start/stop) и отрисовка маркеров.
- `utils.js` — вспомогательные функции.
- `globals.js` — глобальные переменные и DOM-ссылки.

## Советы разработчикам

- При разработке рекомендуется открывать DevTools (Console) и следить за ошибками — некоторые запросы к роутерам (OSRM) выполняются через fetch и могут вернуть ошибки сети.
- Если вы хотите изменить поведение режима маршрута, смотрите `route.js` (функции `startRouteMode`, `stopRouteMode`, `onRouteMapClick`).
- Стили, влияющие на курсор в режиме маршрута, находятся в `style.css` (селектора с `.route-mode-active`).
